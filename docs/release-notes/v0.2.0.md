# ThinkBreak v0.2.0

## 中文

ThinkBreak v0.2.0 将项目重构为跨平台、Hook-first 的等待动作系统。它不再依赖桌面应用，用户可以通过可编辑 Recipe 自己决定 Agent 等待时执行什么。

- 支持 Codex 与 Claude Code 的 `UserPromptSubmit`、`PermissionRequest`、`Stop` 生命周期。
- macOS/Linux 使用 Bash；Windows 提供 PowerShell Dispatcher。
- 默认提供抖音浏览器样板，但不读取 Cookie、密码、Token 或浏览器 Profile。
- Recipe 可以替换为 B 站、小说、音乐、文章、个人网站或任意本地动作。
- 支持 2 秒短任务保护、最新任务 Owner、来源窗口恢复和 30 分钟安全超时。
- 新增 ThinkBreak Skill，让 Codex 或 Claude Code 成为配置界面。
- 配置、状态和用户 Recipe 只保存在本机，不上传任务内容或浏览记录。
- 移除 Swift 菜单栏应用、浏览器控制器、MCP、常驻服务和固定媒体/阅读预设。
- “看广告换模型额度”仍然只是透明的玩梗/兴趣收集方向，不连接广告、账户、Token 或中转服务。

这是未签名、未经 Apple 公证的开源测试版本。Windows 用户请按照 README 配置 PowerShell Hook；当前版本仍建议在真实 Windows 环境中完成一次验证。

## English

ThinkBreak v0.2.0 refactors the project into a cross-platform, Hook-first waiting-action system. It no longer depends on a desktop app: users decide what happens during an Agent task through editable Recipes.

- Supports the `UserPromptSubmit`, `PermissionRequest`, and `Stop` lifecycle events for Codex and Claude Code integrations.
- Bash is provided for macOS/Linux and PowerShell for Windows.
- The default Douyin example opens the user's browser without reading cookies, passwords, tokens, or browser profiles.
- Recipes can be replaced with Bilibili, novels, music, articles, personal websites, or arbitrary local actions.
- Includes the two-second short-task guard, newest-task ownership, source-window restoration, and a 30-minute safety timeout.
- Adds a ThinkBreak Skill so Codex or Claude Code can act as the configuration interface.
- Configuration, transient state, and user Recipes stay on the local machine; task content and browsing history are not uploaded.
- Removes the Swift menu-bar app, browser controller, MCP server, resident service, and fixed media/reading presets.
- “Watch ads for model credits” remains only a transparent joke/interest-collection direction; there is no advertising, account, token, or relay service.

This is an unsigned, non-notarized open-source test release. Windows users should follow the README for PowerShell Hook setup; a real Windows validation pass is still recommended.
